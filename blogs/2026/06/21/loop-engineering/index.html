The job is shifting from typing prompts to building the system that types them. Loop engineering is the practice of moving from prompting an AI to designing the system that prompts it. Instead of treating code generation as a one-shot query, you structure the work as an iterative, self-correcting cycle.

This post covers the anatomy of a good loop, patterns from the Loop Library, and Claude Code’s four native primitives — /loop, Desktop tasks, Routines, and the Monitor tool — along with a decision framework for choosing between them.
